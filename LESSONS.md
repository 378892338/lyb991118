	### 并发安全与记忆合并协议
	如果存在多个 Agent 会话并行工作，必须遵循以下协议：
	1. **独立写入**：每个子 Agent 只能写入自己的临时文件 `.ai/memory/[agent-id]-lessons.md`，严禁直接抢占写入主 `LESSONS.md`。
	2. **主从合并**：由主 Agent 在会话结束或任务合并节点，执行记忆合并操作。
	3. **合并原则**：
	   - 若出现冲突（相同触发场景），保留**效能评分最高**的条目。
	   - 若为互补经验，则整合为一条，并重置召回次数与效能评分。
	   - 合并完成后，删除子 Agent 的临时文件，并确保主 `LESSONS.md` 行数 **≤ 150 行**。

## [LESSON-002] 发现型 Hook 环境变量依赖风险
- **创建**: 2026-06-22
- **类型**: 发现型
- **触发场景**: 60轮飞轮系统设计测试发现 PreToolUse Hook 依赖 `$CLAUDE_TOOL_INPUT` 变量，但该变量在当前环境不存在

## [LESSON-003] 发现型 自动化设计必须区分"强制机制"和"建议机制"
- **创建**: 2026-06-22
- **类型**: 发现型
- **触发场景**: 架构图中标注"自动飞轮闭环"，但实际测试发现 ⑥ 环中仅 ④（SessionStart 召回）为真实自动，其余环节均依赖 AI 自觉执行 prompt 文本

## [LESSON-005] 纠错型 JSON 中 Bash Hook 的 awk 多层转义必然失败
- **创建**: 2026-06-22
- **类型**: 纠错型
- **触发场景**: Stop Hook 命令 `bash -c '... | awk "{if (\\$1>150) ...}'` 经 JSON→bash 多层转义后实际执行报 syntax error near unexpected token `else`
- **正确做法**: JSON 中的 bash hook 禁止使用 awk（引号嵌套不可控）。替代方案：`wc -l < file`（仅输出数字）+ 纯 bash `if [ "$n" -gt N ]`。如必需复杂文本处理，用 `python3 -c` 替代 awk。
- **召回次数**: 1
- **最后召回**: 2026-06-22
- **效能评分**: ✅ (1/1 次有效)
- **状态**: ACTIVE

## [LESSON-006] 发现型 分类器自毒死锁 — Hook 语法错误导致全系统挂起
- **创建**: 2026-06-22
- **类型**: 发现型
- **触发场景**: PreToolUse 中 `echo $CLAUDE_TOOL_INPUT` 未加引号，复杂 Bash 命令中的特殊字符导致 echo 崩溃 → 分类器收到非法 JSON → 判定"不可用" → Write/Edit/Bash 全部被拒绝。形成 "修复 Hook 需修改 settings.json，但修改被分类器阻断" 的死锁。
- **正确做法**: 所有 Hook 变量引用必须加双引号 `echo "$VAR"`。Hook 命令须经 3 层转义链审计 (JSON→bash→子进程) + 实际 dry-run 验证。
- **召回次数**: 0
- **最后召回**: —
- **效能评分**: 0/0 (初始)
- **状态**: ACTIVE
